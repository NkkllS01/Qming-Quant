# OKX 合约策略挖掘路线图

日期：2026-06-03  
适用范围：个人自用 OKX USDT 永续合约，低杠杆，BTC/ETH 优先  
目标：筛选能被当前项目架构落地、可回测、可解释、适合逐步模拟盘验证的策略方向。

> 本文档不构成投资建议。第一阶段的策略目标不是收益最大化，而是验证数据、回测、风控、模拟盘和执行链路是否可靠。

---

## 1. 策略筛选原则

适合当前项目的策略必须满足：

- 主要依赖 OHLCV K 线，当前数据层已经支持。
- 低频或中频优先，避免依赖毫秒级盘口延迟。
- 逻辑可解释，方便定位亏损来自策略、数据、执行还是风控。
- 低杠杆可运行，不依赖极高换手。
- 先在 BTC/ETH 上验证，再扩展到 SOL 等高流动性 USDT 永续合约。
- 策略只产生 `Signal`，不能直接下单，所有开仓必须经过 `PortfolioRiskManager` 和 `SymbolLeaseManager`。

## 3. 第一阶段主策略

### 3.1 ATR / Donchian 通道突破

当前状态：项目已有 `AtrChannelBreakoutStrategy` 雏形。

核心逻辑：

```text
最近 N 根 K 线最高价 = 上轨
最近 N 根 K 线最低价 = 下轨
当前收盘价突破上轨，且 ATR 百分比在合理范围内，开多
止损 = ATR * multiplier
止盈 = reward_risk * stop distance
```

所需数据：

- 15m / 1h OHLCV
- ATR
- 可选：高周期趋势过滤
- 可选：成交量过滤

建议默认实验组合：

```text
symbol: BTC-USDT-SWAP, ETH-USDT-SWAP
timeframe: 15m, 1h
lookback: 20, 40
atr_multiplier: 1.5, 2.0
reward_risk: 2.0, 3.0
min_atr_pct: 0.1%
max_atr_pct: 8%
```

主要风险：

- 横盘行情假突破。
- 突破后马上回落。
- 小币种插针导致异常止损。
- 参数过度优化后实盘失效。

回测重点：

- 交易次数是否足够。
- 手续费和滑点占比。
- 不同年份、月份、震荡期、趋势期的表现。
- `lookback` 和 ATR 倍数轻微变化后结果是否稳定。
- 信号 candle 后一根 K 线开盘成交，避免同 K 线收盘成交偏差。

第一阶段动作：

1. 用 BTC/ETH 15m 和 1h 数据生成基础回测报告。
2. 加入成交量过滤前后对比。
3. 加入高周期趋势过滤前后对比。

### 3.2 多周期 EMA 趋势跟随

核心逻辑：

```text
1h / 4h 判断主趋势
15m 找回调后继续上涨或突破入场
ATR 或结构低点止损
固定风险比例计算仓位
```

示例规则：

```text
如果 1h close > EMA50 且 EMA20 > EMA50，认为多头趋势成立
15m 回调后重新站上 EMA20 或突破前高，产生做多信号
止损 = ATR * 1.5 或最近 swing low
止盈 = 2R / 3R 或移动止损
```

所需数据：

- 15m、1h、4h OHLCV
- EMA
- ATR
- 可选：Funding Rate

主要风险：

- 震荡行情频繁止损。
- 趋势末端追高。
- 参数看起来简单，但仍然容易过拟合。

回测重点：

- 趋势行情与震荡行情分开统计。
- BTC 调参、ETH 验证；ETH 调参、BTC 验证。
- 不追求最优 EMA 参数，重点看参数区间是否稳定。

第一阶段动作：

1. 把当前 starter `MultiTimeframeTrendStrategy` 升级为真实多周期 EMA + ATR 版本。
2. 与 ATR Donchian 突破共用同一套风险和回测指标。

当前实现状态：

- 默认使用 10/20 EMA，便于本地小样本 smoke test 和 CLI 快速跑通。
- 可以通过参数切换到 20/50、30/80 等研究组合。
- 如果 `context["higher_timeframe_candles"]` 存在，会使用高周期 EMA 趋势确认；否则使用当前周期 candles 作为兼容回退。
- 止损和止盈比例由 ATR、`atr_stop_multiplier`、`reward_risk` 计算，策略不直接决定仓位。

## 4. 第二阶段过滤器

### 4.1 成交量确认

用途：减少假突破。

示例规则：

```text
价格突破 Donchian 上轨
当前成交量 > 最近 20 根平均成交量 * 1.3
高周期趋势向上
开多
```

回测重点：

- 过滤后交易次数是否过少。
- 胜率是否提升。
- 盈亏比是否下降。
- 手续费占比是否改善。

### 4.2 Funding Rate 过滤器

用途：识别拥挤交易，避免在极端 funding 下追逐拥挤方向。

建议第一版只记录 funding，不直接参与决策。第二阶段再测试：

```text
极端正 funding: 降低追多仓位或过滤部分追多信号
极端负 funding: 降低追空仓位或过滤部分追空信号
```

注意：Funding 不是反转信号，只能作为拥挤程度参考。

### 4.3 市场状态过滤

用途：区分趋势、震荡、高波动异常环境。

可用指标：

- ATR 百分比
- EMA slope
- ADX
- 布林带宽度
- 最近 N 根 K 线净变化 / 总波动

第一版先做统计标签，后续再决定是否进入策略逻辑。

## 5. 第二阶段策略

### 5.1 波动率过滤均值回归

核心逻辑：

```text
高周期无明显趋势
价格跌破布林带下轨或偏离 EMA 过大
成交量没有异常放大
做多反弹
止损放在极端低点或 ATR 距离
止盈回到均线附近
```

适用范围：

- BTC/ETH 优先
- 15m / 30m / 1h
- 不建议低于 5m

主要风险：

- 单边下跌中不断抄底。
- 合约市场爆仓链条会让“超跌”继续超跌。
- 小币均值回归容易变成接飞刀。

上线前要求：

- 必须有趋势过滤。
- 必须有明确止损。
- 必须统计连续亏损。
- 不在小币上直接使用。

## 6. 第三阶段研究方向

### 6.1 Open Interest 辅助过滤

用途：

```text
价格上涨 + OI 上升 = 趋势可能更真实
价格上涨 + OI 下降 = 可能只是空头平仓
价格横盘 + OI 快速上升 = 可能酝酿大波动
```

需要新增数据：

- OKX open interest 历史
- OI 与 K 线时间对齐
- OI 变化率统计

第一版不要直接用 OI 入场，先做辅助标签和研究报告。

### 6.2 多策略组合轮换

候选组合：

```text
趋势市场: 多周期 EMA 趋势
突破市场: ATR Donchian
震荡市场: 均值回归
高波动异常: 暂停或降仓
```

前置条件：

- SimulationBroker 能恢复持仓。
- 风控能统一约束所有策略。
- 回测能按策略、币种、月份拆分指标。

### 6.3 轻量 ML 过滤器

只建议作为过滤器，不建议直接预测涨跌或直接输出下单。

可研究方向：

- 判断当前信号是否处于高风险市场状态。
- 对已有策略信号打分。
- 识别过热、过冷、波动异常环境。

前置条件：

- 有足够干净的历史数据。
- 有严格样本外验证。
- 有 walk-forward 和过拟合检测。

## 7. 暂不适合第一阶段的方向

高频盘口做市：不适合。它依赖低延迟 WebSocket、盘口深度、订单队列位置、撤单速度和手续费等级，个人账户和当前架构都不适合。

跨交易所套利：不适合。它需要多交易所资金调度、转账风险管理、价格延迟控制、提现限制处理和多账户对账，工程复杂度过高。

复杂 ML / 深度学习：不适合第一阶段。数据量、特征工程、标签设计、样本外验证、漂移检测都很重，容易回测漂亮但实盘失效。

高杠杆网格：谨慎，不作为主策略。合约网格在单边行情、资金费率和爆仓风险下尾部风险很大。

马丁格尔 / 无限补仓：不适合。个人账户尤其应该避免把它作为系统策略方向。

## 8. 策略开发路线

### Phase A：第一阶段闭环策略

目标：验证系统，不追求收益最大化。

任务：

- 完善 `AtrChannelBreakoutStrategy`。
- 升级 `MultiTimeframeTrendStrategy`。
- 回测 BTC/ETH 15m 和 1h。
- 输出总收益、最大回撤、胜率、盈亏比、交易次数、手续费占比、连续亏损次数。
- SimulationBroker 跑固定历史窗口和本地最新 K 线。

### Phase B：过滤器增强

目标：提升信号质量。

任务：

- 加成交量确认。
- 加 ATR/波动率状态过滤。
- 记录 Funding Rate。
- 对比过滤前后交易次数、回撤、胜率和大单收益。

### Phase C：组合和研究

目标：建立可扩展策略研究流程。

任务：

- 增加 MarketRegimeDetector。
- 增加 OI 数据层。
- 做多策略组合统计。
- 做 walk-forward 和参数稳定性报告。

---

## 9. 当前最优结论

第一版最优策略组合：

```text
主策略 1: ATR Donchian 突破
主策略 2: 多周期 EMA 趋势跟随
辅助过滤: ATR 波动率过滤
第二阶段过滤: 成交量、Funding Rate
暂不实现: 高频、跨交易所套利、复杂深度学习、马丁格尔
```

原因：这两个主策略最适合当前工程阶段。它们可以直接使用本地 K 线数据、策略接口、回测、模拟盘和统一风控，而且亏损原因相对容易分析。
